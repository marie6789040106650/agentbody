## 一、结构化理解与核心问题

### 5个核心问题
```
分析所有输入内容并生成 5 个核心问题，当这些问题被回答时，捕捉所有输入内容的主要观点和核心含义。
```
**适用**: 快速抓住材料支点，为后续扩展铺路

### 讲座版核心问题
```
审查所有上传的材料并生成 5 个能够捕捉核心含义的核心问题。
重点关注：核心主题与定义、强调的关键概念、概念之间的关系、提到的实际应用。
```
**适用**: 教学场景或系统梳理

## 二、发现亮点与生成素材

### 有趣片段（Steven Johnson）
```
这些来源中最令人惊讶或最有趣的信息是什么？请包含关键引用。
```
**适用**: 从长文档中快速抽取"可写、可讲"的亮点

### 有趣片段的定向扩展
```
我有兴趣撰写关于 [主题] 的内容。这些来源中与 [主题] 相关的最令人惊讶的事实或想法是什么？请包含关键引用。专注于 [特定方面]，而不是 [其他方面]。
```
**适用**: 定向搜索"新意"，避免浅层重复

## 三、教学与音频生成

### 测验节目（音频概述）
```
一个有两位主持人的测验节目。第一位主持人向第二位提问关于 [主题] 的问题，共 10 个问题，混合多选与判断。主持人有时故意答错，另一位纠正，最后分享结果。
```
**适用**: 课堂互动、音频提要

### 多语言播客
```
这是 Deep Dive 的第一集，完全用 [语言] 进行。特别说明：全程仅使用 [语言]；除了澄清独特术语外不使用英语。
```
**适用**: 在官方支持之前先行生成多语言内容

## 四、角色化分析（官方范式）

### 产品经理（PM）审稿
```
扮演审查内部文档的首席产品经理。无情地扫描可行动洞察，忽略冗余。综合成"决策备忘录"格式：
- 用户证据（直接引用用户问题）
- 可行性检查（提到的技术限制）
- 盲点（源文本中缺失的内容）
使用要点；若问题模糊则要求澄清。
```
**适用**: 把零散信息转化为可决策材料

### 科学研究助理
```
扮演高级科学家的研究助理。语气严格客观、正式、精确；假设具备 [领域] 的高级知识，不定义标准术语。
关注：方法论、数据完整性与矛盾证据；优先样本量、设计与统计显著性。
使用加粗分节：关键发现、方法论优势/劣势、矛盾。
```
**适用**: 学术评阅与方法论审查

### 中学教师
```
扮演富有魅力的中学教师，把源文档翻译成七年级能理解的语言。
结构包括：
- "长话短说"（一句话）
- 类比（现实世界比喻）
- 词汇表（3 个难词的简单定义）
对密集段落拆成判断题测验。
```
**适用**: 科普与跨层级沟通

## 五、综述、矛盾与差距

### 文献综述的主题抽取
```
从关于 [主题] 的论文中识别 5–10 个高频主题；对每个主题给出：
- 简短定义
- 涉及论文（附引用）
- 一句话说明其讨论方式
```
**适用**: 跨文献的主题地图

### 发现矛盾
```
从关于 [主题] 的论文中识别主要矛盾或相互冲突的发现。对每个矛盾给出：
- 各方具体主张（引用与出处）
- 可能的分歧原因
```
**适用**: 立论与讨论的边界清晰化

### 基于来源的差距分析
```
根据我上传的材料分析这次尝试：
- 项目 [我尝试了什么]
- 方法 [我采取的步骤]
- 结果 [发生了什么]
- 预期 [应该发生什么]
与来源交叉引用：
- 我遗漏了哪些步骤？
- 哪些误解导致失败？
- 来源如何具体解决？
```
**适用**: 复盘与纠偏

## 六、从研究到行动

### 实施概念
```
帮助我实施 [主题] 的概念。对每个相关来源：
- 引用关键证据
- 连接到其他检索信息
- 标注冲突观点
- 给出明确行动步骤
综合为有序行动清单，承认知识空白。
```
**适用**: 把理论落地为操作

### 综合概念
```
综合 [主题1] 与 [主题2] 的联系（允许抽象）。对每个来源：
- 引用关键证据
- 连接到其他信息
- 标注冲突
- 记录有趣组合
输出清晰摘要，承认空白。
```
**适用**: 跨域联想与创新

### 综合主题分析（长篇）
```
提供关于 [主题] 的准确、理由充分的信息。
规划：基本方面、关键问题、现有争议。
结构：概述（摘要、要点）
标准：区分事实与解释、用证据支持、保持客观
```
**适用**: 系统性长文与报告

## 七、观点对抗与辩论

### 辩论格式
```
生成两位主持人关于 [主题] 的辩论，持对立观点。
主持人 1 支持 [立场 A]，主持人 2 支持 [立场 B]。
他们应相互挑战，引用来源的具体证据，让听众判断论证强弱。
```
**适用**: 展示分歧、训练批判性思维

---

## Nova 使用原则

1. **不是套用所有提示词** — 根据任务目标与材料类型选择
2. **配合使用** — 多个提示词可以组合形成工作流
3. **最小化步骤，最大化结果** — 不要为了用而用
4. **通用性** — 这些框架适用于所有 AI 工具（ChatGPT/Claude/NotebookLM/秘塔等）

