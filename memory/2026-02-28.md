# 2026-02-28 AgentBody 工作日志

## 项目背景

目标：做一个本地运行的 AI 浏览器自动化工具

## 核心技术

| 技术 | 说明 |
|------|------|
| nodriver | Python 浏览器控制库 |
| FastAPI | HTTP API 服务 |
| Tauri | 桌面应用打包 |

## 关键成果

### 1. HTTP API 服务 (port 9229)

| API | 功能 |
|-----|------|
| `/start` | 启动浏览器 |
| `/smart_open` | 智能打开 URL |
| `/click` | 点击元素 |
| `/click_at` | 坐标点击 |
| `/fill` | 填写表单 |
| `/focus` | 切换标签页 |
| `/screenshot` | 截图 |
| `/get_element_position` | 获取元素坐标 |
| `/task/start` | 开始任务 |
| `/task/end` | 结束任务 |
| `/cookie/save` | 保存 Cookie |
| `/cookie/load` | 加载 Cookie |

### 2. 视觉点击流程（重要突破）

```
1. /smart_open → 打开页面
2. /screenshot → 截图
3. AI 看图识别按钮位置
4. /get_element_position → 获取坐标
5. /click_at → 坐标点击
6. /click 检测 new_tabs → OAuth 新窗口
```

### 3. OAuth 流程测试成功

```
Vercel 登录页 → 点击 GitHub 按钮 → 检测到新窗口 → GitHub OAuth 页面
```

## 经验教训

### 1. 选择器问题
- nodriver 不支持 `:has-text()` 等伪类
- 解决方案：AI 看图 + 坐标点击

### 2. 融合方案 vs CDP
- AI-Native Browser CDP 有 Chrome 路径问题
- 最终选择：AgentBody nodriver + AI 看图

### 3. GitHub 注册问题
- inkblocklab.com 域名被 GitHub 识别为 Google Workspace
- 输入邮箱后自动跳转到 Google OAuth
