## 关键教训总结 (10:30 更新)

### 为什么不能生成 .so 文件？

1. **根本原因**: 缺少 LLVM
2. **原理**:
   ```
   Rust 代码 → LLVM → eBPF 字节码 → .so 文件
                 ↑
           需要 LLVM 支持
   ```
3. **现象**:
   - 没有 LLVM → 生成 .dylib (macOS 原生)
   - 有 LLVM → 生成 .so (eBPF 格式)

### 发现过程

| 时间 | 操作 | 发现 |
|------|------|------|
| 10:03 | anchor build 失败 | Rust 版本问题 |
| 10:05-07 | 尝试修复依赖 | Cargo.lock v4 |
| 10:13 | 学习 Solana_study_code | 项目结构 |
| 10:20 | Rust 1.89.0 构建成功 | 只能生成 dylib |
| 10:20+ | 意识到需要 LLVM | 根本原因 |

### 解决方案

1. 安装 LLVM: `brew install llvm`
2. 配置环境: `export LLVM_SYS_140_PREFIX=$(brew --prefix llvm)`
3. 构建: `cargo build-sbf`
