# Today I Learned - 2026-02-03

## OpenClaw Browser Profile Discovery

### Problem
- Browser Relay extension from daily Chrome was causing conflict with OpenClaw's managed browser
- Error: "Chrome extension relay is running, but no tab is connected"

### Solution
OpenClaw supports **multiple browser profiles** with different CDP ports:

| Profile | Port | Purpose |
|---------|------|---------|
| `openclaw` | 18800 | Managed browser (isolated) |
| `chrome` | 18792 | Extension relay |
| Custom | 18801+ | User-defined |

### Key Insights

1. **No Extension Needed** - OpenClaw's managed browser works without Browser Relay
2. **Isolation** - Completely separate from daily Chrome
3. **Port Isolation** - Each profile uses different CDP port
4. **Default Profile** - Default is `chrome`, must specify `profile="openclaw"`

### Commands

```bash
# Start managed browser
openclaw browser --browser-profile openclaw start

# Check status
openclaw browser --browser-profile openclaw status

# Navigate
openclaw browser --browser-profile openclaw navigate "https://example.com"

# Screenshot
openclaw browser --browser-profile openclaw screenshot
```

### Verified Working
- ✅ CDP port 18800 responding
- ✅ Page navigation works
