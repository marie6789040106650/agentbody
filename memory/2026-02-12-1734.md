assistant: **OpenClaw Managed Browser 支持的操作：**

| 操作 | 说明 |
|------|------|
| `open` | 打开指定 URL |
| `screenshot` | 页面截图 |
| `snapshot` | 获取可访问性树 (DOM 结构) |
| `act` | 执行页面操作：点击、输入、滚动、选择等 |
| `navigate` | 导航到新 URL |
| `console` | 获取控制台日志 |
| `pdf` | 打印为 PDF |
| `upload` | 文件上传 |

**工作原理：**

1. **独立 Chrome 实例** — 启动一个专用的 Chrome 进程，使用独立的用户数据目录 (`~/.openclaw/browser/openclaw/user-data`)，与你的主浏览器完全隔离

2. **Chrome DevTools Protocol (CDP)** — 通过 WebSocket (ws://127.0.0.1:18800) 与 Chrome 通信，控制浏览器行为

3. **无头/可视化模式** — 支持无头模式运行，也可以显示完整的 Chrome 窗口界面

4. **可访问性树** — snapshot 会获取页面的无障碍树 (accessibility tree)，比纯 HTML 更结构化，适合 AI 理解和操作

简单说：就是一个专门给 AI 控制的浏览器，AI 能看到页面结构、截图、点击/输入，就像你手动操作浏览器一样。
