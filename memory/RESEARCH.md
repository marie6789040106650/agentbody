## ⏰ 定时启动机制

### 触发方式

| 优先级 | 触发方式 | 说明 |
|--------|----------|------|
| **高** | Cron 触发 | 定时启动 |
| **中** | 心跳触发 | 心跳时检查并启动 |
| **低** | 手动触发 | 随时可以启动 |

### 遗漏补偿策略

| 时间范围 | 策略 |
|----------|------|
| 当天内 | 心跳时检测并启动 |
| 超过1天 | 立即启动，不等待 |
| 超过3天 | 简化研究，快速完成 |

### 核心原则

> **宁可启动，不要等待**

## 🔧 启动检查流程

```bash
# 检查是否到研究时间
research_day=$(date +%u)  # 1=周一, 4=周四
current_hour=$(date +%H)

if [ "$research_day" -eq 1 ] && [ "$current_hour" -ge 9 ]; then
    start_research
fi

if [ "$research_day" -eq 4 ] && [ "$current_hour" -ge 14 ]; then
    start_research
fi
```
