## AI-Native Browser 项目 - Chromiumoxide 集成

### 背景
- 用户要求坚持使用 AI-Native Browser 作为项目目标
- 目标是实现自动化浏览器控制（注册、登录、部署）

### 关键决策

1. **放弃手写 WebSocket CDP** - 实现复杂且不稳定
2. **采用 chromiumoxide** - 蓝图推荐的 Rust CDP 库
3. **简化实现** - 使用 parking_lot::RwLock 替代 tokio::sync

### 技术问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| Async/Await 与 Axum 不兼容 | trait 实现复杂 | 改用同步操作 |
| RwLock 无法在 async 外使用 | tokio runtime 限制 | parking_lot |
| act_handler 阻塞 | async 调用 | 完全同步化 |

### 最终实现

- `cdp/mod.rs`: 使用 `parking_lot::RwLock` 的简化 CdpClient
- 所有操作都是同步的 (`launch()`, `navigate()`, `get_url()`)
- 与 axum handler 兼容

### 2026-02-27 下午更新 - 真正的 Chromiumoxide 集成

**技术方案**:
- 独立线程 + tokio runtime：浏览器在专用线程中运行，每个线程有自己的 tokio runtime
- mpsc channel：主线程通过 channel 与浏览器线程通信
- parking_lot::RwLock：同步状态共享

**解决的问题**:
- chromiumoxide 完全 async → 线程化隔离
- BrowserConfigBuilder moves self → 重新赋值
- evaluate_expression 参数 → 从 `chromiumoxide_cdp::cdp::js_protocol::runtime` 导入
- get_title 返回 Result<Option<String>> → ok().flatten().unwrap_or_default()
