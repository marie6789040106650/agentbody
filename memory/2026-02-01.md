### Problem:
- OpenClaw is undergoing a migration from Clawdbot
- The `minimax-portal-auth` extension still references the old `clawdbot/plugin-sdk`
- This causes a module loading error: "Cannot find module 'clawdbot/plugin-sdk'"
- Error occurs when attempting to use MiniMax models

### Temporary Fix Applied:
- Modified `/Users/bypasser/.nvm/versions/node/v22.22.0/lib/node_modules/openclaw/extensions/minimax-portal-auth/index.ts`
- Replaced `import { emptyPluginConfigSchema } from "clawdbot/plugin-sdk"` 
- With direct imports from `@sinclair/typebox` and recreated the function locally
- This allows the extension to load without the deprecated dependency

### Code Change:
```typescript
// Before:
import { emptyPluginConfigSchema } from "clawdbot/plugin-sdk";

// After:
// Import from the new OpenClaw plugin system instead of the deprecated clawdbot/plugin-sdk
import { Type } from "@sinclair/typebox";

// Create equivalent function for backward compatibility
const emptyPluginConfigSchema = () => Type.Object({});
```

### Status:
- This is a temporary workaround for the migration issue
- Should be revisited when OpenClaw officially addresses the Clawdbot to OpenClaw migration
- Need to monitor future OpenClaw updates to see if this issue is properly resolved
- May need to revert this change once official fix is released

## 浏览器操作方式 (自动选择)

### 3种操作方式

| 方式 | Profile | 隔离性 | 扩展支持 | 适用场景 |
|------|---------|--------|----------|----------|
| **OpenClaw 托管** | `openclaw` | ✅ 完全独立 | ✅ 需手动安装 | 主要自动化操作 |
| **Chrome 扩展接管** | `chrome` | ❌ 共享配置 | ✅ 现有扩展 | 需要已登录账号 |
| **无头模式** | `openclaw` + `headless=true` | ✅ 完全独立 | ❌ 无界面 | CI/CD、后端任务 |

### 自动选择逻辑

**默认使用 `openclaw` profile：**
- 独立环境，不干扰日常 Chrome
- 可安装专用扩展
- 需要截图、交互操作时

**使用 `chrome` profile：**
- 需要访问已登录账号时
- 需要使用现有 Chrome 扩展时

**使用无头模式：**
- 后端任务、不需要界面时
- 追求更快速度时
